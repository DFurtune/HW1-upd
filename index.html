GIT

git config --global user.name blabla

встановити name користувача

git config --global user.email blabal@bla

встановити email користувача

git init

створити новий репозиторій

git status

показує стан файлів у робочих директоріях: котрі файли оновлено, але не додано до індексу; котрі очікують комміту в індексі

git add

додати всі змінені файли (до stage)

git commit

створити новий коміт

git checkout

копіює файли з історії в робочу директорію, можна переключатися між гілками

git branch

показати всі гілки

git push

вивантажує вміст локального репозиторію у віддаленний

git pull

завантажує вміст віддаленного репозиторію

git merge

з'єднати 2 гілки в одну

git clone

клонує віддалений репозиторій

.gitignore файл

це простий текстовий файл, в кожному рядку якого міститься шаблон файлу або каталогу, який необхідно проігнорувати

git revert

Скасовує внесені в комміті зміни і додає новий комміт з отриманим вмістом

git reset

Скасовує зміни. Змінює вказівник HEAD і, опціонально, стан індексу. Також може змінити файли в робочій директорії параметром --hard

git rebase

спосіб перенести зміни з одної гілки в іншу

git cherry-pick

використовується для перенесення окремих коммітів з одного місця репозиторію в інше

git stash

призначена для того, щоб помістити поточні зміни, які ми виконали у файлах, в окреме сховище, та повернути файли до вихідного стану
